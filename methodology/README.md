# Methodology Index

This directory contains project methodology rather than canonical theory.

Methodology governs how Project FAR research is conducted, tested, audited, revised, and promoted.

Canonical definitions and theorems belong in `../theory/`; examples belong in `../examples/`; exploratory notes belong in `../research/`.

## Governing Methodology

- `research-doctrine.md` — canonical research doctrine for honesty, traceability, promotion, rejection, and revision.
- `foundational-discovery-protocol.md` — canonical protocol for grounding Project FAR from first principles before deriving framework architecture.

## Areas

- `proof-standard/` — proof standards, canonicalization, candidate generation, dependency graphs, and primitive evaluation procedures.
- `validation/` — validation methodology and domain validation cases.
- `falsification/` — falsification methods and negative tests.
- `comparison/` — comparison methodology.

## Methodology Boundary

Methodology documents define research process.

They do not by themselves establish canonical theory, accepted primitives, accepted necessities, or completed theorems.

## Archived Methodology Material

Superseded methodology documents are retained under `../archive/superseded/methodology/` when they contain meaningful historical content.
