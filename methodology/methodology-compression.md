# Methodology Compression Program

## Status

Research Program

## Purpose

The Methodology Compression Program exists to minimize the independent commitments of Project FAR's methodology without reducing its ability to expose defects, improve artifacts, or support reproducible investigations.

The optimization objective is:

> Maximum methodological rigor with minimum independent commitments.

## Core Principle

Every methodological rule must justify its own existence.

A rule that can be derived from more general principles should be recorded as a derived consequence rather than retained as an independent methodological commitment.

## Compression Procedure

Every methodology rule shall be evaluated using the following questions:

1. What concrete failure does this rule prevent?
2. What evidence demonstrates that this failure actually occurs?
3. Can another rule already prevent the same failure?
4. Can this rule be derived from a more general rule?
5. If the rule were removed, what repository artifacts or investigations would fail?
6. Does the rule introduce unnecessary complexity?
7. Does the rule improve independent evaluation?

A rule that cannot justify its continued existence becomes a candidate for removal, merger, or derivation.

## Candidate Rule Register

Each methodology rule should eventually record:

- purpose;
- prevented failure mode;
- supporting audit evidence;
- derivable status;
- dependency relations;
- removal impact;
- current recommendation.

## Compression Outcomes

Each rule investigation ends in one of the following states:

- retained as independent;
- derived from stronger rule(s);
- merged;
- replaced;
- removed;
- under investigation.

## Candidate Generators

The long-term objective is to discover a minimal generating set for the methodology.

Examples of candidate generators include:

- Explicitness sufficient for independent evaluation.
- Independent evaluation.
- Minimum justified commitment.

These are research hypotheses, not canonical axioms.

## Repository Policy

No new methodology rule should be canonized until:

- it survives artifact audits;
- it addresses a demonstrated failure mode;
- it cannot be satisfactorily derived from existing methodological commitments.

## Success Criterion

The compression program succeeds when the methodology becomes simultaneously:

- smaller;
- easier to audit;
- equally or more capable of exposing structural defects;
- easier for independent investigators to apply consistently.

Methodological growth is not itself evidence of methodological improvement.
