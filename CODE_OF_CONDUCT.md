# Code of Conduct

## Purpose

Project FAR is a collaborative research project.

This Code of Conduct establishes the standards of behavior expected from all contributors and participants.

The objective is to foster productive, evidence-based collaboration while maintaining the integrity of the project.

---

# Expected Conduct

Contributors are expected to:

- Treat others with professionalism and respect.
- Critique ideas rather than individuals.
- Support claims with clear reasoning or evidence whenever possible.
- Remain open to correction and revision.
- Welcome constructive criticism.
- Keep discussions relevant to the project.
- Help maintain a collaborative environment.

---

# Research Standards

Discussion within Project FAR should emphasize:

- Logical consistency
- Explicit reasoning
- Clear definitions
- Intellectual honesty
- Constructive disagreement

Disagreement is an expected and valuable part of the research process.

The goal of discussion is to improve the framework rather than defend personal positions.

---

# Unacceptable Conduct

The following behaviors are not acceptable:

- Personal attacks or harassment.
- Discrimination or hateful conduct.
- Deliberate misrepresentation of another contributor's position.
- Repeated bad-faith participation.
- Spam or intentionally disruptive behavior.
- Knowingly introducing misleading or fraudulent contributions.

---

# Resolving Disagreements

Contributors are encouraged to resolve disagreements by:

1. Clarifying definitions.
2. Identifying assumptions.
3. Presenting explicit arguments.
4. Providing supporting evidence where appropriate.
5. Remaining focused on the substance of the discussion.

Whenever possible, disagreements should be documented so that future contributors can understand the reasoning behind important decisions.

---

# Scope

This Code of Conduct applies to:

- GitHub Issues
- Pull Requests
- Discussions
- Project documentation
- Any other official Project FAR collaboration spaces

---

# Enforcement

Project maintainers are responsible for interpreting and enforcing this Code of Conduct.

Actions may include:

- Requesting revisions.
- Moderating discussions.
- Removing inappropriate content.
- Restricting participation when necessary.

Enforcement should be proportionate, transparent whenever practical, and directed toward maintaining a productive research environment.

---

# Guiding Principle

Project FAR exists to improve the understanding of reasoning through rigorous, explicit, and collaborative inquiry.

Every contributor shares responsibility for maintaining an environment that supports that objective.
