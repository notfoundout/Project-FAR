# Scope

## Purpose

This document defines the formal scope of Project FAR.

It specifies the class of reasoning processes that Project FAR is intended to represent, analyze, and evaluate.

All claims concerning the universality, sufficiency, and applicability of the framework are relative to the scope defined herein.

---

# Scope

Project FAR is a formal framework for representing, analyzing, and evaluating explicit representations of reasoning.

Project FAR does not model cognition, consciousness, intelligence, or the internal mechanisms by which reasoning is produced.

Instead, it operates upon reasoning once that reasoning has been made explicit through representation.

Accordingly, the primary object of study within Project FAR is the explicit representation of a reasoning process.

---

# Reasoning Process

For the purposes of Project FAR, a reasoning process is a process that establishes, evaluates, classifies, or resolves one or more candidates by means of justified transitions between representations.

This definition is structural.

It is independent of:

- subject matter;
- implementation;
- physical medium;
- cognitive mechanism; and
- the identity of the reasoning agent.

The formal representation of reasoning processes is defined separately by the primitive architecture of Project FAR.

---

# Explicit Representation

A reasoning process falls within the scope of Project FAR only if it admits an explicit representation.

An explicit representation is a representation whose content is objectively identifiable independently of the internal cognitive state that produced it and is sufficiently specified to permit formal analysis.

Explicit representations may include, but are not limited to:

- written language;
- spoken language;
- symbolic notation;
- diagrams;
- formal proofs;
- structured data; and
- computational representations.

The physical medium of the representation is irrelevant.

Only objective accessibility and formal analyzability are required.

---

# Membership Criterion

A process belongs within the scope of Project FAR if and only if all of the following conditions are satisfied.

1. The process constitutes a reasoning process.

2. The reasoning process admits an explicit representation.

3. The explicit representation is sufficiently specified to permit formal analysis within Project FAR.

Processes failing to satisfy any of these conditions lie outside the scope of the framework.

---

# Domain Independence

Project FAR is domain-independent.

Its applicability does not depend upon the discipline in which the reasoning occurs.

Accordingly, the framework is intended to encompass reasoning arising in domains including, but not limited to:

- mathematics;
- science;
- law;
- history;
- philosophy;
- politics;
- engineering; and
- everyday practical reasoning.

These examples illustrate the intended breadth of the framework but do not define its scope.

---

# Exclusions

Project FAR does not presently encompass processes that do not constitute explicit representations of reasoning.

Examples include:

- reflexive behavior;
- purely random processes;
- physical causation independent of reasoning;
- raw sensory perception; and
- emotional experience independent of explicit reasoning.

These exclusions apply only insofar as such processes lack explicit representational reasoning.

---

# Universality

The claim that Project FAR is universal is always relative to the scope defined in this document.

Accordingly, universality means that every reasoning process within the stated scope can, in principle, be represented, analyzed, and evaluated by Project FAR.

Project FAR makes no claims regarding processes outside its stated scope.

---

# Relationship to the Primitive Architecture

The primitive architecture provides the formal representation of reasoning processes that fall within the scope defined in this document.

The architecture is justified independently through the theoretical development of Project FAR.

Accordingly, the scope defines what Project FAR studies, while the primitive architecture defines how those reasoning processes are formally represented.

---

# Design Principle

Project FAR is defined by structural admissibility rather than by subject matter.

Any reasoning process satisfying the membership criterion belongs within the scope of the framework regardless of its origin, discipline, implementation, or physical realization.
