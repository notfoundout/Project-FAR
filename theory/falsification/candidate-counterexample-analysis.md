# Candidate Counterexample Analysis

Status: Provisional analysis for v0.2.0.

## Purpose

The falsification harness identifies fixtures that strain FAR's primitive architecture. A `candidate counterexample` classification is not itself a successful falsification. It marks a case requiring analysis.

This document evaluates the current candidate counterexamples:

- paradoxical reasoning;
- inconsistent calculus;
- opaque intuition or oracle reasoning.

Each case is classified as one of:

- `real counterexample`;
- `conservative extension`;
- `outside FAR scope`.

---

## Analysis Standard

A candidate becomes a real counterexample only if all of the following hold:

1. the system is explicit reasoning;
2. the system cannot be represented using FAR's current primitives;
3. the missing element cannot be derived through the registry of derived concepts;
4. the missing element cannot be handled by a conservative extension;
5. the system requires a genuinely new primitive.

If the case can be represented by adding a domain-specific calculus, interpretation policy, constraint, or derived concept while leaving the primitives intact, it is a conservative extension.

If the case lacks inspectable representations, transitions, or rules, it may fall outside FAR's scope as currently defined: explicit reasoning.

---

## Candidate 1 — Paradoxical Reasoning

Fixture: `examples/far/reasoning-systems/paradox.far.yaml`

### Problem

The fixture represents a liar-style paradox: a sentence denies its own truth status. Under ordinary bivalent interpretation, assigning either true or false generates instability.

### Stress Point

The stress point is not representation. FAR can represent:

- the sentence;
- the self-referential relation;
- the contradiction generated by a bivalent interpretation;
- the need for a special semantic policy.

The stress point is semantic stability. FAR's ordinary satisfaction vocabulary may be too weak unless the interpretation explicitly permits partial, paraconsistent, stratified, fixed-point, or rejection-based semantics.

### Analysis

This does not yet force a new primitive. The primitive `Interpretation` already carries semantic assignment. The primitive `Reasoning Calculus` already carries admissible transition rules. A paradox-handling policy can be modeled as a specific interpretation/calculus pair.

What FAR needs is not a new primitive, but registered derived concepts for semantic-instability handling.

### Decision

Classification: `conservative extension`

Reason: paradox requires a specialized semantic policy, not a new primitive beyond representation, interpretation, and calculus.

### Registry Resolution

Handled by:

- `D-027` — Semantic Instability;
- `D-028` — Guarded Self-Reference;
- `D-029` — Paraconsistent Calculus, where a paradox is handled through a non-classical calculus.

## Candidate 2 — Inconsistent Calculus

Fixture: `examples/far/reasoning-systems/inconsistent-calculus.far.yaml`

### Problem

The fixture represents a calculus where both `A` and `not A` are asserted while explosion is not automatically permitted.

### Stress Point

The stress point is whether contradiction automatically destroys reasoning or whether a calculus can preserve controlled inference under inconsistency.

### Analysis

FAR can represent:

- `A`;
- `not A`;
- the contradiction relation;
- a policy blocking explosion;
- a calculus governing which transitions remain admissible.

This is directly expressible through `Representation`, `Representational Structure`, `Interpretation`, and `Reasoning Calculus`.

The missing machinery is a domain-specific calculus policy. That is a conservative extension, not a primitive failure.

### Decision

Classification: `conservative extension`

Reason: non-explosive inconsistency can be represented as a calculus-level constraint.

### Registry Resolution

Handled by:

- `D-029` — Paraconsistent Calculus;
- `D-030` — Non-Explosive Inference.

## Candidate 3 — Opaque Intuition or Oracle Reasoning

Fixture: `examples/far/reasoning-systems/opaque-oracle-reasoning.far.yaml`

### Problem

The fixture represents a conclusion asserted by an oracle or opaque intuition without an accessible derivation, rule, or transition.

### Stress Point

The stress point is whether an unsupported assertion counts as reasoning. FAR can represent the oracle output as a representation, but if no transition, rule, interpretation, or inferential standard is accessible, there is no explicit reasoning process to evaluate.

### Analysis

If the oracle's internal process is inaccessible, FAR can represent only:

- the assertion;
- the absence of accessible derivation;
- the act of accepting or rejecting the assertion.

That does not make the hidden process explicit. It means the case is outside FAR's current scope unless the oracle supplies inspectable representations and transitions.

The fixture therefore does not falsify FAR. It clarifies FAR's boundary condition: FAR evaluates explicit reasoning, not opaque assertion.

### Decision

Classification: `outside FAR scope`

Reason: without accessible representations, rules, or transitions, the case is not explicit reasoning under FAR's current scope.

### Registry Resolution

Handled by:

- `D-031` — Explicit-Reasoning Scope Boundary;
- `D-032` — Opaque Assertion.

## Current Result

None of the current candidate counterexamples is established as a real counterexample.

The strongest remaining pressure is opaque oracle reasoning, because it forces a boundary decision: either FAR only evaluates explicit reasoning, or FAR must be expanded to model inaccessible cognitive or oracle processes. Under the current Project FAR scope, opaque oracle reasoning is outside scope rather than a successful falsification.
