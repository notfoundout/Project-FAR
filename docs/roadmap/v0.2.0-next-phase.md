# Branch Discipline

New theory, tooling, or verification work should follow this workflow:

1. create a feature branch;
2. make scoped changes;
3. run verification;
4. open a pull request;
5. merge only after CI passes.

Direct commits to `main` should be reserved for emergency fixes or administrative repository changes.
