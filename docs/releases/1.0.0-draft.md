# Project FAR 1.0.0 — Draft Release Notes

Status: unreleased. These notes summarize the cumulative development path from 0.4.0 to the intended 1.0.0 release. They do not authorize publication by themselves.

## What changed since 0.4.0

Project FAR moved from an early formal-vocabulary research repository into a consolidated research and verification system with explicit claim boundaries, mechanized validation, deterministic decision adjudication, external-trace ingestion, regression comparison, evidence manifests, and staged external validation.

## Research and theory

- separated universality into coverage, faithful preservation, operational adequacy, necessity, minimality, reproducibility, and nontrivial constraint;
- separated existential sufficiency from reproducible sufficiency;
- separated local ablation-based necessity from global necessity relative to an explicitly searched vocabulary space;
- adopted preservation vectors, canonical intermediate representation requirements, commitment-equivalence safeguards, and Pareto comparison rules;
- defined the representation protocol as decision-theoretic rather than inferential;
- registered evaluator competence, independence, calibration, and blind-mapping requirements;
- classified additional reasoning domains conservatively, preserving unresolved and extension-required results rather than forcing positive conclusions.

## Repository and governance

- established `main` as the canonical integration line;
- documented the roles of FAR, FARA, FARE, FARM, and FARO;
- classified canonical, transitional, experimental, and archived packages;
- recorded convergence rules for work stranded on diverged branches;
- separated theory, mechanization, verification, product code, and external-validation artifacts;
- prohibited infrastructure expansion from substituting for evidence of operational need.

## Verification and mechanization

- repaired and hardened the canonical intermediate representation and validation paths;
- integrated repository health, formal artifact validation, Lean mechanization, specification export, mutation testing, legacy-oracle comparison, and signed assurance bundles;
- added exact commit and tree binding to merge-authority evidence;
- retained explicit Unknown and Unverifiable outcomes rather than collapsing missing evidence into failure or success.

## Decision integrity

- added the canonical decision package and deterministic validation pipeline;
- added four-state adjudication: `justified`, `unsupported`, `underdetermined`, and `unverifiable`;
- preserved evidence provenance, dependency declarations, unknowns, and incompleteness;
- added deterministic reports and stable command-line entry points;
- added regression comparison for authorization expansion, integrity degradation, restrictions, and reasoning changes;
- added SHA-256 evidence manifests and tamper detection.

## External traces

- added a generic external-trace format;
- added SWE-agent trajectory ingestion;
- preserved source locations and provenance classes;
- prevented inferred dependencies, hidden reasoning, and external truth from being promoted into observed evidence;
- preserved unknown event and semantic completeness.

Because the benchmark outcome was known before the candidate freeze, Candidate 001 does not support a blind detection-value claim.

This supports conservative staged-pipeline adequacy, not broad predictive accuracy.

## Canonical 1.0 interface

The intended 1.0 interface freezes:

- the FAR/FARA/FARE/FARM/FARO role definitions;
- `far-decision`, `far-reasoning-regression`, and `far-external-trace` as supported CLI names;
- the registered decision-package, report, trace, regression, and evidence-manifest schemas;
- the four-state adjudication semantics;
- explicit positive claims and prohibited overclaims.

## Corrections and superseded approaches

- the public baseline for these cumulative notes is `0.4.0`, not `0.3.1`;
- human or organizational independence is not claimed from multiple implementations produced by one agent;
- the abandoned participant-registration path was replaced with isolated computational implementation replication;
- successful mappings are not treated as proof of universality, necessity, or minimality;
- ordinary agent traces are not treated as complete records of hidden reasoning or all relevant events;
- negative, unresolved, and unverifiable external results are retained as valid findings.

## What 1.0.0 will claim

Project FAR can ingest supported external execution artifacts, preserve provenance and uncertainty, compile them into canonical decision packages, apply deterministic operational adjudication, compare result sets, and produce reproducible evidence-bound reports.

## What 1.0.0 will not claim

Project FAR 1.0.0 will not claim unrestricted universality, complete access to hidden reasoning, general benchmark prediction, regulatory certification, or proof that the current vocabulary is globally necessary and minimal.

## Remaining release gates

- clean-install golden reproduction must pass from built artifacts;
- cumulative release notes must be reviewed against repository history beginning at `0.4.0`;
- package metadata and documentation must agree with the frozen public interface;
- the exact release commit must pass every required validation and assurance workflow;
- the release gate must explicitly permit publication before the `1.0.0` tag and GitHub release are created.
