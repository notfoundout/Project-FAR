# Project FAR 1.0 Claim Boundary

Project FAR 1.0.0 may claim that the repository provides a deterministic, auditable path for representing supported reasoning artifacts, preserving explicit provenance and uncertainty, validating decision packages, comparing reasoning corpora, and compiling supported external traces into evidence-bound reports.

## Positive claims permitted

Project FAR 1.0.0 may state that it can:

1. validate supported FAR artifacts and schemas;
2. preserve observed, derived, inferred, declared, and unverifiable evidence distinctions;
3. adjudicate supported decision packages into `justified`, `unsupported`, `underdetermined`, or `unverifiable` states;
4. compare baseline and candidate reasoning corpora for authorization expansion, integrity degradation, restrictions, and reasoning changes;
5. bind reports and source artifacts with deterministic SHA-256 manifests;
6. compile supported SWE-agent trajectories into FAR decision packages without treating hidden reasoning or external truth as observed evidence;
7. reproduce Candidate 001 and Candidate 002 execution paths deterministically;
8. preserve `unverifiable` when ordinary traces do not establish event or semantic completeness.

## Claims not permitted

Project FAR 1.0.0 must not claim:

- unrestricted universality across every reasoning system;
- proof that the current vocabulary is globally necessary or minimal;
- recovery of hidden model reasoning from ordinary traces;
- universal semantic completeness;
- general failure-detection accuracy from Candidate 001 or Candidate 002;
- regulatory certification or production safety assurance;
- independent human or organizational replication where only isolated computational implementations were used;
- that an unresolved benchmark outcome proves FAR predicted the failure.

## External validation interpretation

Candidate 001 validated the acquisition and execution pipeline but was outcome-contaminated before freeze. Candidate 002 preserved blind stage ordering and produced a pre-outcome `unverifiable` result before revealing an unresolved benchmark outcome. Together they support pipeline adequacy and conservative evidence handling. They do not establish broad predictive accuracy.

## Release rule

Any release note, README statement, website copy, abstract, or product description associated with 1.0.0 must remain within this boundary. Stronger claims require separately documented evidence and an explicit amendment to this file.
