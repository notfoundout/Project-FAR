# Reasoning and Contrast Scope v1.0

## Status

Frozen prospective scope and admission rules. Execution remains open.

Identifier: `RCS-001`.

## Purpose

Universal-structure discovery requires positive and contrast classes whose membership is not defined by FARA compatibility or by possession of the candidate structure under test.

## Positive reasoning families

The first discovery program must sample formally specified instances from materially different families:

- deductive proof;
- probabilistic inference;
- defeasible and nonmonotonic reasoning;
- planning and search;
- diagnosis and explanation;
- legal or policy reasoning;
- analogical reasoning;
- distributed institutional reasoning;
- self-modifying machine reasoning;
- human expert reasoning under an explicit observation contract;
- partially observed reasoning;
- semantic or conceptual revision.

Inclusion requires an independently defended reasoning classification, an explicit source contract, and a declared observation or formalization boundary.

## Contrast families

The contrast program must include structured systems not admitted merely because they lack FARA labels:

- passive databases;
- event logs;
- ordinary workflow engines;
- arbitrary labeled transition systems;
- lookup tables;
- traffic controllers;
- cellular automata;
- random finite-state machines;
- copying and serialization systems;
- scripted pipelines;
- disputed borderline processes.

## Admission independence

A system may not enter the positive class because it satisfies FARA, PB-001, a candidate invariant, or the expected conclusion.

A system may not enter the contrast class solely because it fails FARA or lacks the candidate invariant.

## Candidate-neutral observations

For each system the program records:

- observable or formal states;
- transitions and constraints;
- represented distinctions;
- declared semantics, if any;
- history and path dependence;
- objectives or externally imposed tasks;
- grounds or dependencies, if any;
- commitment-like or alternative-like structure, if any;
- uncertainty and revision, if any;
- provenance and correspondence limits.

The record may state that a field is absent, inapplicable, or unresolved.

## Decision use

A candidate universal structure is weakened when:

- an admitted reasoning system lacks it under all admissible reconstructions;
- it appears only under one vocabulary;
- it occurs equally in the contrast class without a reasoning-specific role;
- it can be eliminated without loss of the frozen reasoning commitments.

A contrast occurrence does not automatically refute necessity. It may show that the candidate is a general systems property rather than a reasoning discriminator.

## Nonclaims

This scope does not establish that any listed family is represented adequately, that every instance is reasoning, or that the candidate classes are exhaustive.
