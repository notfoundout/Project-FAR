# EVC-W1 External Proof Reviewer Guide v1.0

## Purpose

This guide supports an eligible external review of the bounded post-W5 Universal-Structure Discovery result. It does not ask the reviewer to endorse Project FAR, FARA, or any universal-structure claim. The reviewer is asked to determine whether the frozen evidence supports the narrower registered conclusion.

## Frozen conclusion under review

Within the frozen benchmark and candidate universe, FARA and LTS-PROV are successful, non-equivalent, mutually incomparable bounded minima. The evidence does not establish one universal kernel, global necessity, global minimality, actual-process correspondence, or external validation.

## Independence requirements

The reviewer must not have constructed the reviewed mappings, proofs, validators, fixtures, comparisons, or synthesis. Running the same artifacts through another agent instance or another implementation controlled by the author does not satisfy this condition.

Before reading substantive evidence, record identity or persistent identifier, affiliation, relevant expertise, conflicts, package release identifier, access time, and whether substantive author contact occurred.

## Review order

1. Read the protocol and package manifest.
2. Inspect the source-scope freezes and negative controls before reading winner-oriented synthesis prose.
3. Evaluate USD-W1 preservation results.
4. Evaluate the USD-W2 candidate comparison and complete cost ledgers.
5. Evaluate USD-W3 invariance transformations and rejected non-equivalences.
6. Evaluate USD-W4 ablation, reconstruction, and equivalent-reintroduction tests.
7. Evaluate USD-W5 equivalence and minimal-frontier classification.
8. Evaluate USD-W6 only as internal robustness, not external independence.
9. Inspect the terminal synthesis and compare every conclusion against the weakest supporting gate.
10. File the initial verdict before receiving substantive author response.

## Mandatory questions

Use `W1-Q01` through `W1-Q10` in the result template, in the same order as the mandatory questions in the protocol. Each answer must be `pass`, `qualified`, `fail`, or `unresolved` and must cite exact artifacts and locations.

## Findings

Every finding must include:

- a unique identifier;
- severity;
- artifact and exact location;
- a reproducible argument, counterexample, or documentary basis;
- affected proof obligation or decision rule;
- effect on the bounded conclusion;
- whether author clarification could resolve ambiguity without changing the frozen evidence.

A blocking finding is not limited to a syntactic proof error. It also includes an omitted required cost, invalid commitment-equivalence, scope selected to fit a winner, preservation failure, fabricated independence, or a counterexample that changes the minimal frontier.

## Verdicts

`confirmed` requires all mandatory questions to pass and no blocking finding.

`confirmed_with_nonblocking_qualifications` requires no blocking finding but preserves at least one material qualification.

`blocking_objection` requires a supported blocking finding. The reviewer need not prove the opposite global theorem.

`unresolved` is appropriate when package completeness, reviewer eligibility, or technical ambiguity prevents a justified terminal decision.

## Author interaction

The initial report must be preserved before author response. Author replies may clarify references or contest reasoning, but they cannot silently alter the reviewed package. A correction to frozen evidence creates a new package version. The original report and initial verdict remain part of the record even if the reviewer later amends the verdict.

## Claim boundary

Completion of this guide, protocol, manifest, or result template is preparation for external review. External validation occurs only after an eligible reviewer independently accesses a frozen release and files an attributable result satisfying the protocol.
