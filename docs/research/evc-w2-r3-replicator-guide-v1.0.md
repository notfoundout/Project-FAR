# EVC-W2 R3 Independent Technical Replication Guide v1.0

## Purpose

This package supports an independent technical replication of the bounded Project FAR universal-structure discovery results. It does not ask the replicator to endorse the theory. It asks whether a separately implemented pipeline, controlled by a separate human or organization, reconstructs the registered USD-W2 through USD-W5 classifications and rejects the registered adversarial mutations.

## What counts as independent

The replication implementation must not be written by contributors to the original executable pipeline. A new prompt, branch, language, or agent session under the same authorial control is not R3 independence. The replicator must control design choices, implementation, execution, and reporting.

The replicator may use the frozen specifications, fixtures, candidate definitions, cost ledgers, equivalence rules, and result schemas. The replicator may not import, translate, or wrap the original executable implementation. Original reference outputs remain sealed until the independent implementation commit is frozen.

## Required sequence

1. Record the immutable package identity and first-access timestamp.
2. Submit the eligibility and conflict declaration.
3. Record an implementation plan and environment.
4. Implement the parser, preservation checks, equivalence checks, cost accounting, Pareto comparison, invariance rules, ablation rules, and minimum-frontier classification independently.
5. Freeze the implementation commit.
6. Run the baseline reconstruction in a clean environment.
7. Run the seven registered mutations separately.
8. Unseal the reference terminal outputs.
9. Compare terminal classifications and registered witnesses.
10. Publish the initial result before receiving substantive author response.

## Mandatory targets

The replication must reconstruct the W2 candidate frontier, W3 bounded invariance result, W4 bounded local-necessity result, W5 multiple-incomparable-minima result, and the terminal bounded-incomparable-kernels synthesis.

A matching terminal label without the required witness structure is insufficient. The replicator should retain enough evidence to show how candidate preservation, charged machinery, translation failure, dominance, and incomparability were calculated.

## Mutation campaign

The mutation corpus tests whether the implementation catches plausible claim or accounting corruption while remaining syntactically valid. It includes equivalence collapse, hidden-cost deletion, global-claim inflation, local-to-global necessity inflation, unique-minimum promotion, counterexample deletion, and independence-label inflation.

A generic exception is not adequate when a mutation is schema-valid. The result should identify the violated gate or provide an equally specific diagnosis.

## Ambiguity and missing evidence

Do not ask the authors for an informal replacement artifact. Record missing evidence as a package-completeness failure. Interpretive ambiguity should be resolved by a declared conservative rule or should make the affected target unresolved.

All author contact must be logged. A substantive package repair after first access creates a new package version; it cannot silently modify the reviewed release.

## Outcomes

`replicated` requires all mandatory targets to match, every mandatory mutation to be detected, complete provenance, deterministic reruns, and no disqualifying dependence.

`partially_replicated` records a mixed result without erasing successful or failed targets.

`not_replicated` records a material classification discrepancy or accepted adversarial mutation after a valid independent execution.

`unresolved` applies when independence, package completeness, determinism, or ambiguity prevents a defensible verdict.

## Claim boundary

R3 replication concerns technical reproducibility of the frozen bounded program. It does not establish conceptual validity, candidate exhaustiveness, cross-context validity, actual-process correspondence, necessity, uniqueness, or universal structure.
