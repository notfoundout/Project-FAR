# Audit — USD-W6 independence

## Layer separation

The package keeps independent proof review, R3 technical replication, R4 adversarial conceptual replication, R5 cross-context replication, and internal multi-implementation robustness as distinct evidence classes.

## Honesty check

No CI runner, signed artifact, isolated compiler, separate verifier, or deterministic comparison is labeled human or organizational independence. The executed result is explicitly internal implementation replication.

## Isolation check

The three internal implementations have no access to each other's generated artifacts. A separate verifier performs normalized deterministic comparison and mutation checks.

## Adversarial check

The package rejects mutations that erase hidden costs, collapse non-equivalent vocabularies, introduce future information, promote a unique minimum, or relabel internal robustness as external verification.

## External gate

Independent verification remains blocked until eligible external reviewers or replicators complete a frozen package without retroactive repair. Reviewer identity, eligibility, prior access, objections, and resolution history must be recorded.

## Claim boundary

The correct result is `internal_robustness_only`. This audit does not establish independent review, external replication, conceptual correctness, cross-context validity, or universal structure.
