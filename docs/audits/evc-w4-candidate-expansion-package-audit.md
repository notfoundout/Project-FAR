# EVC-W4 Candidate Expansion Package Audit

The package is frozen, unexecuted, and unreleased. No materially new candidate has been submitted, admitted, frozen, or evaluated.

The protocol prevents four common false advances: counting a renamed winner as a new architecture; importing winner machinery without cost; changing primitives after observing failures; and treating an unsuccessful bounded search as proof of exhaustiveness.

Admission is candidate-independent. Every candidate must expose its ontology, operations, semantics, dependencies, history, revision rules, and admissibility commitments before benchmark execution. Novelty requires more than different labels or factorization and must survive low-cost bidirectional commitment-equivalence analysis.

Evaluation reuses the frozen preservation, cost, equivalence, invariance, ablation, negative-control, and frontier rules. It does not create a scalar ranking. Failed candidates and failed controls remain visible. Substantive repair creates a versioned new candidate rather than rewriting history.

The package can support a bounded frontier update after execution. It cannot establish an exhaustive architecture universe, global minimality, unique structure, universal structure, or actual-process correspondence.
