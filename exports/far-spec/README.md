# Project FAR Specification Export

`exports/far-spec-v1/` is the only supported machine-readable integration surface for external consumers such as FAR-Verify. External integrations must depend on the exported manifest, checksums, grammar, schemas, semantics, terminology, compatibility policy, and examples rather than on Project FAR internal implementation paths.

## Architecture

The export is generated by `python tools/export_specification.py`. The exporter copies canonical Project FAR artifacts without rewriting theory, proofs, semantics, definitions, or research conclusions. Generated files are limited to export metadata, checksum metadata, example indexing, and the compatibility policy.

## Versioning

The v1 export uses semantic versioning (`MAJOR.MINOR.PATCH`) for `export_version`, `exporter_version`, `schema_version`, and `compatibility_version`. A fixed major version preserves the public integration contract recorded in `manifest.json` and `checksums.json`.

## Reproducibility

The exporter removes and regenerates `exports/far-spec-v1/` deterministically. Repository metadata is read from Git for a new export and preserved from the committed manifest during regeneration so repeated runs produce byte-identical artifacts while retaining the recorded source snapshot metadata.

## Consuming the export

Consumers should:

1. Read `manifest.json`.
2. Verify `checksums.json`.
3. Load only artifacts listed in the manifest.
4. Treat unexported repository files and implementation details as unsupported.
