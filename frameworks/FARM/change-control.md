# FARM Change Control

## Purpose

This document defines FARM change-control gates for stable Project FAR layers.

It does not authorize changes by itself.

---

## Change-Control Scope

Change control applies to stable or frozen layers:

- FARE Mathematics v0.1;
- FAR v1.0 Stable;
- FARO v1.0 Stable;
- any later stable FARA, FARM, or project-level canonical layer.

---

## Change Request

A change request is a recorded proposal to alter a stable artifact.

---

## Required Fields

A change request should identify:

- change identifier;
- source requirement or defect;
- affected artifact;
- affected framework layer;
- proposed change;
- justification;
- expected impact;
- dependencies affected;
- review status;
- decision.

---

## Review Statuses

A change request may be:

- proposed;
- under review;
- accepted;
- rejected;
- deferred;
- superseded.

---

## Gate Rule

No stable-layer change should occur without:

1. a recorded defect or requirement;
2. routing to the correct layer;
3. impact assessment;
4. explicit review decision.

---

## Boundary Rule

FARM records and governs the change-control procedure.

It does not itself change stable framework content.
