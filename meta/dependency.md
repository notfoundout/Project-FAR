# Dependency

## Purpose

This document defines dependency within the meta-theory of Project FAR.

Dependency specifies when one object requires another for its definition, existence, derivation, interpretation, or justification.

It provides the formal basis for analyzing the structure of reasoning and the relationships among concepts within the framework.

---

# Definition

An object **A** depends upon an object **B** if A cannot be established, interpreted, or justified without B under the specified criterion of dependency.

Dependency is always relative to an explicitly identified context.

---

# Relative Nature of Dependency

Dependency is not absolute.

Every statement of dependency shall identify:

- the dependent object;
- the supporting object;
- the criterion of dependency; and
- the context in which the dependency is asserted.

---

# Directionality

Dependency is directional.

If A depends upon B, it does not necessarily follow that B depends upon A.

Accordingly, dependency defines an asymmetric relationship unless otherwise demonstrated.

---

# Dependency Relations

Project FAR recognizes multiple forms of dependency.

Examples include:

- definitional dependency;
- derivational dependency;
- structural dependency;
- semantic dependency;
- justificatory dependency.

Additional dependency relations may be introduced where formally justified.

---

# Definitional Dependency

An object exhibits definitional dependency if its definition requires reference to another object.

---

# Derivational Dependency

An object exhibits derivational dependency if it can be formally derived from another object or collection of objects.

---

# Structural Dependency

An object exhibits structural dependency if its organization requires another object.

---

# Semantic Dependency

An object exhibits semantic dependency if its interpretation requires another object.

---

# Justificatory Dependency

An object exhibits justificatory dependency if its acceptance requires another object as supporting justification.

---

# Dependency Graphs

Dependencies may be represented as directed graphs.

Vertices represent objects.

Directed edges represent dependency relations.

Dependency graphs provide a formal means of analyzing the organization of reasoning.

---

# Cyclic Dependency

A cyclic dependency exists when a sequence of dependency relations returns to its point of origin.

Cycles are not inherently invalid.

However, any cycle used to justify itself without independent support constitutes circular justification.

The acceptability of cyclic structures therefore depends upon the criterion of dependency under investigation.

---

# Independence

Two objects are independent if neither depends upon the other according to the specified dependency criterion.

Independence should not be confused with non-equivalence or non-identity.

---

# Relationship to Primitive Architecture

Primitive concepts should exhibit no derivational dependency upon the remaining primitive architecture.

Establishing this property is a principal objective of T-MINIMAL-PRIMITIVE-ARCHITECTURE.

Derived concepts, by contrast, are expected to exhibit explicit dependency upon previously established concepts.

---

# Relationship to FARO

Many FARO operations require dependency analysis.

Examples include:

- tracing conclusions;
- identifying assumptions;
- detecting circular justification;
- locating unsupported claims;
- simplifying reasoning structures.

Accordingly, dependency is a foundational concept for the operational framework.

---

# Current Research

The following questions remain under investigation.

- How should dependency graphs be formally represented?
- Under what conditions are cyclic dependencies admissible?
- What algorithms best identify minimal dependency sets?
- Can every derivation be represented as a dependency graph?
