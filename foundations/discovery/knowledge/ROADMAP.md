# Knowledge Layer Roadmap

## Purpose

Define the staged development plan for the Knowledge Layer.

## Stage 1 — Pilot (Current)

Completed:

- Knowledge architecture (KA-001)
- Pilot claims
- Pilot evidence
- Pilot hypothesis
- Pilot open question
- Pilot knowledge graph
- Pilot traceability policy

Goal:

Determine whether epistemic objects improve repository traceability.

## Stage 2 — Validation

Objectives:

- Populate knowledge objects only from completed investigations and audits.
- Use the Knowledge Graph during artifact audits.
- Evaluate maintenance cost versus traceability benefit.

Success Criteria:

- Knowledge objects remain synchronized with investigations.
- Audits become more precise through explicit knowledge links.

## Stage 3 — Canonization Decision

Questions:

- Should the Knowledge Layer become canonical?
- Which object types are required?
- Which remain experimental?

No canonization should occur without evidence from Stage 2.

## Guiding Principle

The Knowledge Layer exists to organize justified knowledge, not to increase document count.
