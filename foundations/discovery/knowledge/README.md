# Knowledge Layer

## Status

Pilot Epistemic Object System

## Purpose

This directory contains epistemic objects produced or supported by Project FAR investigations, audits, grounding passes, and methodology work.

The Knowledge Layer is distinct from repository storage.

A claim is not the file that records it.

Evidence is not the investigation that produced it.

A counterexample is not the document that describes it.

## Object Families

- `claims/` — explicit knowledge claims and their support relations.
- `evidence/` — evidence objects supporting, weakening, or challenging claims.
- `counterexamples/` — counterexamples and attempted counterexamples.
- `hypotheses/` — active hypotheses not yet supported enough to become claims.
- `theorems/` — formal theorem-like objects and proof status records.
- `open-questions/` — unresolved questions that structure future inquiry.

## Current Policy

This is a pilot system.

Do not bulk-populate it.

Create knowledge objects only when they are supported by existing investigations or audits.

No knowledge object is canonical merely because it appears here.

## Required Traceability

Every knowledge object should identify:

- its statement or question;
- its status;
- its origin;
- supporting investigations;
- supporting audits or evidence;
- counterexamples or challenges;
- dependencies;
- open questions.
