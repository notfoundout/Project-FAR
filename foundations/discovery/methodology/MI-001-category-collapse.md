# MI-001 — Category Collapse as a Methodological Failure Mode

## Status
Active Methodology Investigation

## Objective
Determine whether category collapse is a recurring methodological failure mode that warrants explicit treatment within Project FAR.

## Evidence Base
This investigation is based on GP-001 through GP-012.

## Working Definition
Category collapse occurs when two or more conceptually distinct roles are treated as though they were the same concept, obscuring dependencies, evaluation, or interpretation.

## Observed Cases
- Object vs Representation
- Representation vs Interpretation
- Interpretation vs Meaning
- Rule vs Execution
- Process vs Record
- Model vs Theory
- Theory vs Framework
- Framework vs Methodology
- Framework vs Architecture
- Investigation vs Repository Artifact

## Questions
1. Is category collapse an independent failure mode?
2. Can it be derived from Explicitness or Orthogonality?
3. Does testing for category collapse improve audits?
4. Does it explain a substantial proportion of defects found during grounding?

## Falsification Criteria
The hypothesis fails if:
- most observed cases are better explained by existing methodological principles;
- category separation does not improve audits;
- the concept cannot be operationalized consistently.

## Success Criteria
The hypothesis is supported if:
- independent investigations repeatedly detect category collapse;
- separating categories materially improves precision;
- the pattern is reproducible across unrelated artifact families;
- the principle cannot be fully derived from existing methodology.

## Current Status
Evidence is sufficient to justify investigation but not canonization.
No methodological revision is recommended until dedicated artifact audits test this hypothesis.
