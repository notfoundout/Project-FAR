# Investigation I1-DEPENDENCY — Discovery of Dependency

Previous Identifier: I1

## Status

In Progress

---

# Purpose

Discover the necessary and sufficient conditions under which one accepted component depends upon another.

This investigation does not assume any dependency exists.

It seeks to discover the criterion by which dependencies are objectively established.

---

# Research Question

Under what conditions does a dependency exist between two accepted components?

---

# Scope

This investigation concerns dependency as it applies to the accepted results of Project FAR.

It does not assume any existing repository architecture.

It does not assume any ordering of components.

It does not assume that every component participates in a dependency.

---

# Candidate Criterion

A proposed dependency between components A and B shall be represented as:

A → B

The dependency exists if and only if the accepted construction or acceptance of A necessarily requires B.

This criterion is a hypothesis and has not yet been accepted.

---

# Discovery Procedure

For each proposed dependency:

1. State the dependency.
2. State the claimed necessity.
3. Attempt to eliminate the dependency.
4. Attempt to construct A without B.
5. Record the result.
6. If construction succeeds, reject the dependency.
7. If construction fails, determine whether the failure necessarily results from the absence of B.
8. Accept the dependency only if no successful elimination is discovered.

---

# Acceptance Criterion

This investigation succeeds when a criterion is discovered that objectively distinguishes dependencies from non-dependencies.

---

# Failure Criterion

This investigation remains incomplete if:

- multiple incompatible dependency criteria remain equally supported;
- no criterion distinguishes dependencies from non-dependencies;
- the criterion depends upon undefined concepts.

---

# Current Status

In Progress.

# Investigation — Requirement

## Status

In Progress

---

# Purpose

Determine whether the concept of requirement is primitive, derivable, or eliminable.

---

# Research Question

What must be true for one accepted component to require another?

---

# Observed Problem

The current candidate dependency criterion states:

> A depends upon B if A necessarily requires B.

The term "requires" has not been accepted.

Accordingly,

the criterion contains an undefined concept.

No dependency criterion containing undefined concepts can presently be accepted.

---

# Candidate Hypotheses

## H1

Requirement is primitive.

---

## H2

Requirement is derivable from accepted concepts.

---

## H3

Requirement is eliminable and should not appear in the dependency criterion.

---

# Discovery Procedure

For each hypothesis:

1. Attempt construction.
2. Attempt reduction.
3. Attempt elimination.
4. Record the result.

Reject every hypothesis contradicted by the accepted results.

---

# Acceptance Criterion

This investigation succeeds when one and only one hypothesis remains consistent with the accepted results.

---

# Failure Criterion

This investigation remains incomplete if:

- multiple hypotheses remain equally supported;
- every hypothesis fails;
- additional undefined concepts are introduced.

---

# Current Status

In Progress.
